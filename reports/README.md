## Report Types

### Progress Reports
Document what was completed, issues encountered, and next steps.

### Audit Reports
Results from cleanup audits, accessibility checks, performance analysis.

### Sprint Summaries
High-level overview of sprint accomplishments and metrics.

### Incident Reports
Documentation of bugs, issues, or unexpected behavior.

## Guidelines

- Date all reports
- Keep reports factual and concise
- Include actionable items
- Link to relevant code or documentation
- Archive completed work
